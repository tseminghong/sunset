Storage: SSD vs HDD

SSD (Solid State Drive)

  • Faster data accessing speed
  • Resistant to shock
  • Less power consumption
  • Smaller physical size
  • Suitable for use as external portable drive

HDD (Hard Disk Drive)

  • Lower cost per unit storage
  • Easier to recover data (in some cases)

Memory

Stores the initial computer startup program (bootup program, program for the fundamental operations of the hardware).

  • Non-volatile: Can store data permanently, even without power.
  • Difficult to rewrite (Read-Only nature, though some types like EEPROM can be rewritten).
  • Direct access (Corrected from sequential).
  • Permanent storage for firmware.
  • Relatively small storage capacity.
  • Data remains when power is lost.

Used for processing programs and data currently in use (e.g., word processors, web browsers, games).

  • Volatile: Data is lost when power is turned off.
  • Erasable and rewritable.
  • Temporary storage for active tasks.
  • Supports both read and write operations.
  • Random access / direct access for fast data retrieval.

Stores the most frequently used data and instructions for quick access by the CPU.

  • Small in size compared to RAM.
  • Located very close to or on the processor.
  • Volatile.
  • Speeds up data access for the CPU, improving performance.
  • Extremely fast access times.

An extension of RAM, utilizing hard disk space to simulate additional main memory.

  • Allows the system to run larger programs or more programs simultaneously than physical RAM alone would allow.
  • Data is swapped between RAM and the disk (paging/swapping).
  • Slower performance if the computer relies too heavily on it, due to slower disk access speeds compared to RAM.

Loader

The loader is a crucial part of the operating system responsible for preparing a program for execution.

  • Loads the program from secondary storage (like an HDD or SSD) into main memory (RAM).
  • Sets the Program Counter (PC) register with the memory address of the first instruction of the program, so the CPU knows where to begin execution.

    • Multi-Core Processors / Multiple Processors

    Modern CPUs often have multiple cores, or a system might have multiple physical processors. This enables:

    • Parallel Processing: Multiple tasks or parts of a task execute simultaneously on different cores/processors.
    • Multi-threading: A single process can have multiple threads of execution running concurrently.
    • Multitasking: The ability to run several programs or processes seemingly at the same time by rapidly switching between them.
    • Pipelining: An instruction execution technique where multiple instructions are overlapped in execution, like an assembly line.

    CPU Components

    • CU (Control Unit): Directs and coordinates most of the operations in the computer. It interprets instructions and generates control signals.
    • ALU (Arithmetic and Logic Unit): Performs arithmetic (addition, subtraction) and logical (AND, OR, NOT) operations.
    • Registers: Small, high-speed storage locations within the CPU used to temporarily hold data and instructions. Key registers include:
      • IR (Instruction Register): Holds the current instruction being executed.
      • ACC (Accumulator): Stores intermediate arithmetic and logic results.
      • PC (Program Counter): Holds the memory address of the next instruction to be fetched.
      • MAR (Memory Address Register): Holds the memory address of data to be fetched or stored.
      • MDR (Memory Data Register): Holds the data being transferred to or from memory.

    CPU Cycle (Fetch-Decode-Execute-Store)

    1. Fetch: Retrieve the next instruction from memory (RAM) whose address is in the Program Counter (PC).
    2. Decode: The Control Unit (CU) interprets the instruction to determine what operation needs to be performed.
    3. Execute: The Arithmetic Logic Unit (ALU) performs the specified operation (e.g., arithmetic calculation, logical comparison, data movement).
    4. Store: The results of the execution are saved in registers or written back to main memory.

    CPU Performance Factors

    • Clock Rate (Clock Speed): Measured in Gigahertz (GHz), it indicates how many cycles the CPU can perform per second. Higher is generally faster.
    • Number of Cores: Multiple cores allow the CPU to handle more tasks simultaneously (parallel processing).
    • Cache Size/Speed: Larger and faster cache memory reduces the time spent waiting for data from RAM.
    • Instruction Set Architecture (ISA): The design of the instructions the CPU can execute.
    • Bus Speed/Width: Affects how quickly data can be moved between the CPU and other components.

Pointing Devices for Accurate Drawing

These devices allow for precise input, especially useful in graphics applications.

  • Digitizer (or Graphics Tablet): A flat surface on which the user "draws" with a stylus. The tablet translates the stylus movements into digital signals that the computer can interpret. Offers high precision and pressure sensitivity.
  • Graphic Tablet: Often used interchangeably with Digitizer. It's a specific type of digitizer.
